新Linux子系统规划
2020-07-06

最近饱尝旧子系统各种弊病的痛击，终于下定决心重装一遍子系统

子系统不就是给人带来轻松体验的吗？只要打开cmder再输入bash就能轻松使用，相比于开机半分钟的虚拟机简直好到不行有木有！！然而随着使用子系统一年多的时间我也渐渐发现，子系统同样存在一些问题，如没有完善的UI（现在虽然有UI了，但现在电脑c盘空间不够，等新电脑在折腾吧），层出不穷的小BUG以及占用c盘空间等问题。

本次重装子系统主要诱因是python pip的疯狂报错问题，深层原因是过去对Linux系统的认识过于浅显，以至于各种软件过于不规整，以及系统中由于年少无知下载的anaconda始终如鲠在喉，威胁着以后软件的安装（第一次有了喜欢的系统，又碰见了刚好能用的软件，两份快乐叠加在一起，却产生了anaconda的报错，为什么，会变成这样呢）

在此基础上，我更换了Ubuntu18.04，并对新子系统进行了以下规划：

系统位置：C:\Users\lizai\AppData\Local\Packages\CanonicalGroupLimited.Ubuntu18.04onWindows_79rhkp1fndgsc\LocalState\rootfs

根目录：lzh@双语之城:/$ ls
bin boot dev etc home init lib lib64 media mnt opt proc root run sbin snap srv sys tmp usr var

其中opt中新设biotools文件夹，用于存放安装的软件
